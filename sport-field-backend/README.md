## рџ› пёЏ Setup

1) **Import your SQL file** into a database (e.g., MySQL):
```
mysql -u root -p < your_schema.sql
```
Or use a GUI (MySQL Workbench, DBeaver, etc.)

2) **Copy env** and set DB + JWT secrets:
```
cp .env.example .env
```
Edit `.env`:
```
DATABASE_URL="mysql://root:password@localhost:3306/sportfield_db"
JWT_ACCESS_SECRET="your_access_secret"
JWT_REFRESH_SECRET="your_refresh_secret"
CORS_ORIGIN="http://localhost:3000"
PORT=8080
```

3) **Install deps**:
```
npm install
```

4) **Prisma introspect + generate** (creates prisma/schema.prisma automatically):
```
npx prisma db pull
npx prisma generate
```

5) **Run dev server**:
```
npm run dev
```
Backend will run at: **http://localhost:${PORT or 8080}**

## рџ”ђ Auth
- `POST /api/auth/register` вЂ” email, password, name, role (optional; defaults to 'user')
- `POST /api/auth/login` вЂ” returns accessToken & refreshToken
- `POST /api/auth/refresh` вЂ” get new access token
- `GET /api/users/profile` вЂ” get current user info (requires Authorization: Bearer <token>)

## рџЏџпёЏ Fields
- `GET /api/fields` вЂ” list fields (filter by sport/location optional)
- `GET /api/fields/:id` вЂ” field details
- `POST /api/fields` вЂ” (owner) create field
- `PUT /api/fields/:id` вЂ” (owner) update field
- `DELETE /api/fields/:id` вЂ” (owner/admin) delete field

## рџ“… Bookings
- `POST /api/bookings` вЂ” create booking (user)
- `GET /api/bookings/user/me` вЂ” my bookings (user)
- `PATCH /api/bookings/:id/status` вЂ” approve/reject (owner)
- `DELETE /api/bookings/:id` вЂ” cancel booking (user/owner according to policy)

## рџ’і Payments
- `POST /api/payments/create` вЂ” create payment intent (mock/provider-ready)
- `GET /api/payments/:id` вЂ” get payment detail
- `POST /api/payments/webhook` вЂ” provider callback endpoint

## рџ›  Admin
- `GET /api/admin/dashboard`
- `GET /api/admin/users`
- `PATCH /api/admin/users/:id/status`
